Стили оформления карточек

Если хочется использовать какие-то неуниверсальные стили,

(так называемые Vendor Prefix — приставки производителя)
cтили, зависящие от браузера, — то в Anki это -webkit-

не -moz-, не -ms- и не -o-.

Те, кто знают CSS, привыкли, что
шрифты для браузеров можно задавать списком через запятую:
нет первого, а вдруг есть второй, нет второго — может есть третий и так далее.

В Anki это не работает!

Здесь можно указать только один шрифт, можно даже не заключать его название в апострофы или кавычки. Если указанный шрифт не установлен в системе — всё, Finita la commedia, сливай вода, приехали! берётся шрифт по умолчанию, дальнейшие указания через запятую не рассматриваются!

Ограничения JavaScript

В Anki у javascript:

  • внешние файлы <script src="_myFile.js">

    • должны быть в каталоге collection.media

    • и их имена должны начинаться с _подчёркивания,
      чтобы эти файлы экспортировались вместе с колодами.

  • нет отладчика (консоли)

  • document.write не работает на сайте AnkiWeb

  • alert и т.п. функций нет в некоторых мобильных клиентах

  • не поддерживаются localStorage (локальное хранилище HTML5)

  • и куки (document.cookie)

Так что остаётся только работа в элементами DOM в чистом виде.

Классы тега body

Вместо document.writeln(…​); можно использовать такой код, например:
<script>
 function echo(id, html) {
    var sp = document.getElementById(id);
    (sp != null) && (1 != sp.nodeType && (sp = sp.previousSibling),
    (sp != null) && (sp.innerHTML = html.replace(/ /g,'<br>')));
 }
</script>

<div id="inferno" style="border-bottom:dashed 1px silver;"></div><script>echo('inferno', document.documentElement.className);</script>
<div id="info"></div><script>setTimeout(function(){echo('info', document.body.className);},1);</script>
Запуск по таймауту через 1 миллисекунду гарантирует,
что код будет выполнен только когда вся страница будет уже готова.
Дело в том, что в момент формирования страницы
классы тегу <body> ещё не назначены, просто ничего не будет показано,
в отличие от классов тега <html>

Фон для метки

Как сделать разный фон для карточек в зависимости от тегов(меток)?

Я понимаю что меток может быть несколько.
Но хотелось бы визуально отличать разные карточки,
так как я использую как правило только по одной метке на карточку.

В деле различения карточек от записей с разными метками возможны два подхода.

Первый:

просто показывать все метки, что есть у записи.

<div id=Tags>{{Tags}}</div>
#Tags { background-color: #cfc; overflow: hidden;
 position: fixed; bottom: 0px; left: 0px; right: 0px;
 border-top: solid 1px green; height: 20px;
 color: darkgreen; font-size: 14px;
}

Второй:

создать классы, одноимённые меткам.

Код для вставки на лицевую сторону карточки
<script>
setTimeout(function(){var klassList = '{{Tags}}'.split(' ');
for (var i = 0; i < klassList.length; i++) {
 document.body.classList.add(klassList[i]);
}},1);
</script>

Сами классы описать в секции стилей как обычно.

Если на оборотной стороне не используется {{FrontSide}} , то

Код для вставки на оборотную сторону карточки
<script>
var klassList = '{{Tags}}'.split(' ');
for (var i = 0; i < klassList.length; i++) {
 document.body.classList.add(klassList[i]);
}
</script>

Да, на оборотной стороне прокатывает работает и без таймаута.

Можно скачать колоду с примером.

SEPARATOR

С иерархическими тегами проблема

В разделителе из двойного двоеточия по умолчанию —
двоеточие в имени класса не допускается.

# Separator used between hierarchies
SEPARATOR = '::'
В настройках дополнения его надо заменить

На _подчёркивание, например.

# Разделитель, используемый между уровнями
#SEPARATOR = '::'
SEPARATOR = '_'

 

Маленькие хитрости

По центру карточки

Иногда хочется, чтобы содержимое карточки
  не прижималось к верхнему краю,
    а располагалось по центру окна.

html { display: table; height: 100%; width: 100%; }
body { display: table-cell; vertical-align: middle; }

Тут есть нюанс:
4тобы так помещать текст в центр по вертикали, лицевая и оборотная стороны карточки должны быть одинаковой высоты. Иначе перемещение вопроса слегка вверх при появлении ответа доставляет изрядно заметный дискомфорт, который, в свою очередь, значительно понижает результативность запоминания   (так верстают только мудаки).
Общий вывод — овчинка выделки не стоит.

Особенно с учётом того, что под такое выравнивание по высоте приходится довольно существенно перепахивать дорабатывать как шаблоны сторон карточек, так и их стили.

Всё-таки смотреть в верхнюю часть экрана комфортнее, чем в его центр, прежде всего для шеи.

Ну а чтобы к верхнему краю окна не прижималось,
достаточно body.card{padding-top:1em;}

Без черты

Если вам хочется сохранять работоспособность кнопки Перевернуть,
но при этом совсем не хочется видеть горизонтальную черту на своих карточках,
то вместо удаления текста <hr id=answer> из шаблонов оборотных сторон карточек
можно просто через стили с глаз долой скрывать черту из видимости:

hr#answer { visibility: hidden; }

Использование hr#answer { display: none; } приведёт к тому,
что вопрос и ответ будут показаны рядом на одной строке
(разумеется, это для случая разметки по умолчанию,
в которой поля не заключены в блочные элементы).

Финт ушами

Если колоды никуда не вложены (находятся на верхнем уровне)
и в их именах отсутствуют пробелы (да и другие спец. символы)
/скажем, они заменены на подчёркивания/,
а в этих колодах работают однотипные карточки,
то можно оформлять их по-разному в зависимости от колоды:

<div class="{{Deck}}">{{Вопрос}}</div>
.Колода_1 { font-weight: bold; }
.Колода_2 { font-style: italic; }

Что характерно: в предпросмотре редактора шаблонов эта фишка не работает почему-то, только в предпросмотре Обозревателя Anki либо уже при показе карточки.
